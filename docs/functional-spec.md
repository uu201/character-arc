# CharacterArc 功能文档

- 文档版本：`v0.1`
- 更新时间：`2026-04-28`
- 产品形态：`Electron 桌面应用`
- 技术方向：`Electron + Vue 3 + Naive UI + TypeScript`
- 原型来源：[prototypeDesign/UI.html](d:/JavaProjects/MuMuAINovel-main/CharacterArc/prototypeDesign/UI.html)

## 1. 项目目标

CharacterArc 是一款本地优先的 AI 小说创作桌面应用，面向需要管理项目、设定、大纲与正文的创作者。

首版目标不是扩展功能边界，而是：

- 一比一复刻现有原型的页面结构、视觉层次和主要交互
- 在原型基础上补齐真实可用的数据、状态与本地存储能力
- 支持主题色切换，且不破坏原型的整体风格

## 2. 开发原则

- 原型优先：首版页面信息架构、布局比例、交互入口以原型为准
- 克制扩展：未在原型中出现的复杂模块不进入首发
- 本地优先：项目、章节、设置保存在本地
- 安全隔离：API Key 等敏感信息仅保存在 Electron 主进程侧
- 可验证：每个页面都必须有清晰的交互结果和验收标准

## 3. 设计约束

### 3.1 一比一复刻范围

需要尽可能还原以下内容：

- 双视图结构：`项目中心` / `工作台`
- 工作台的信息架构、页面层级与命名
- 左侧导航、主内容区、章节创作页的结构关系
- 卡片、按钮、输入框、标签、弹层的风格基调
- 编辑器页的阅读宽度、留白、顶部工具区
- 新建项目向导模态框的结构与字段顺序

允许做的工程化调整：

- 用 Naive UI 组件替换原型中的原生控件
- 为适配桌面应用加入真实数据状态、加载态、空状态、错误提示
- 用设计令牌统一颜色、阴影、圆角、字号

### 3.2 主题色切换

首版必须支持主题色切换。

要求如下：

- 默认主题为原型中的蓝色主色系
- 至少支持 `蓝色 / 绿色 / 橙色 / 玫红色` 4 套主题色
- 主题切换只替换语义色，不改变布局和组件结构
- 主题切换后以下元素需同步变化：
  - 主按钮
  - 激活导航项
  - 标签高亮态
  - 输入框聚焦态
  - AI 相关强调色
  - 章节激活圆点
- 用户选择的主题色需要本地持久化

### 4.1 一级视图

- 工作台

### 4.2 工作台核心面板

- 世界观设定
- 角色图鉴
- 大纲规划
- 章节编辑器
- 项目与应用设置
- 独立 AI 创作助理窗口
- 左侧章节列表与导航区

### 4.3 全局模态框

- 新建项目向导

## 5. 功能清单

## 5.1 项目中心

对应原型区域：项目列表首页。

### 功能说明

- 展示项目卡片列表
- 支持点击项目进入工作台
- 支持点击“新建项目”打开向导
- 顶部保留页面标题与主操作按钮

### 首版字段

- 项目标题
- 项目封面
- 题材分类
- 当前字数
- 最近编辑时间

### 首版交互

- 点击项目卡片：进入该项目工作台
- 点击“新建项目”：弹出新建项目向导

### 验收标准

- 项目卡片布局与原型视觉一致
- 至少支持 1 个真实项目对象渲染
- 无项目时提供空状态

## 5.2 新建项目向导

对应原型区域：`wizard-modal`。

### 功能说明

用于快速创建新项目，并决定是否自动生成初始内容。

### 表单字段

- 作品标题
- 题材分类
- 目标字数
- 核心点子 / 简介
- 是否自动调用 AI 生成初始世界观与大纲

### 首版交互

- 点击取消：关闭模态框
- 点击创建项目：
  - 校验必填字段
  - 新建项目
  - 若勾选自动生成，则进入工作台后触发初始化任务
  - 默认进入新项目工作台

### 验收标准

- 表单布局、字段顺序与原型一致
- 创建成功后可在项目中心再次看到新项目

## 5.3 工作台整体框架

对应原型区域：`view-workbench`。

### 功能说明

工作台是项目内主工作区，采用双层工作模式：

- 常规工作台：`左侧导航 + 中间主内容`
- 章节创作页：独立写作工作台
- AI 助手：作为当前章节工作台的独立子窗口打开

### 首版要求

- 工作台保持单项目上下文
- 世界观 / 角色 / 大纲 / 设置在主工作台内切换
- 章节创作进入独立写作模式
- AI 助手支持打开、关闭，并与当前章节上下文同步
- 支持从项目中心返回

## 5.4 左侧导航与章节列表

对应原型区域：`sidebar`。

### 功能说明

作为工作台一级导航与章节入口。

### 导航项

- 世界观管理
- 角色管理
- 大纲规划
- 项目设置

### 章节区

- 展示项目章节列表
- 当前章节高亮
- 点击章节切换编辑器内容

### 首版交互

- 点击导航项：切换中间面板
- 点击章节项：进入对应章节编辑器
- 点击首页按钮：返回项目中心

### 验收标准

- 高亮态、激活圆点、留白与原型一致
- 章节列表支持滚动

## 5.5 世界观设定

对应原型区域：`panel-worldview`。

### 功能说明

用于管理项目世界观条目。

### 条目字段

- 标题
- 内容
- 排序
- 创建时间
- 更新时间

### 首版能力

- 查看世界观条目卡片
- 新增条目
- 编辑条目
- 删除条目
- 自动记录排序、创建时间、更新时间

### UI 要求

- 使用卡片式布局
- 卡片头部展示标题与编辑入口
- 内容区支持长文本显示

### 验收标准

- 至少支持新增、编辑、删除、列表展示完整闭环

## 5.6 角色图鉴

对应原型区域：`panel-characters`。

### 功能说明

用于管理角色卡片与基础人设信息。

### 角色字段

- 姓名
- 身份/定位
- 标签列表
- 角色简介
- 头像/色块占位

### 首版能力

- 查看角色卡片列表
- 搜索角色
- 新建角色
- 编辑角色
- 删除角色
- AI 生成角色草案

### UI 要求

- 保持三段式卡片信息：头像、标题/标签、简介
- 搜索框、AI 生成、新建按钮位置与原型一致

### 验收标准

- 搜索可按角色名筛选
- AI 生成功能至少支持生成一条可落库角色草案

## 5.7 大纲规划

对应原型区域：`panel-outline`。

### 功能说明

用于按卷和章节管理故事大纲。

### 大纲字段

- 所属卷名
- 卷目标字数
- 章节标题
- 预估字数
- 核心冲突
- 剧情描述
- 排序

### 首版能力

- 展示卷标题
- 展示章节节点列表
- 新增章节节点
- 编辑章节节点
- 删除章节节点
- AI 扩写大纲

### UI 要求

- 中间内容宽度、卷标题位置、章节节点卡片结构与原型一致
- “新增章节节点”保留虚线占位样式

### 验收标准

- 支持至少一卷多章结构
- 章节节点可增删改排序
- 支持拖拽调整节点顺序

## 5.8 章节编辑器

对应原型区域：`panel-editor`。

### 功能说明

作为正文创作主界面，用于编辑章节标题与正文内容。

### 章节字段

- 章节标题
- 所属卷
- 章节序号
- 正文内容
- 更新时间
- 字数统计

### 首版能力

- 编辑章节标题
- 编辑正文
- 手动保存
- 自动保存
- 切换章节时加载对应内容
- 历史版本入口

### 首版历史版本要求

- 每次手动保存生成一个版本快照
- 自动保存仅更新当前草稿，不强制生成版本
- 支持查看版本列表
- 支持恢复指定版本

### UI 要求

- 中央稿纸式排版与原型一致
- 正文区域保留大留白、高行距、专注阅读感

### 验收标准

- 编辑内容可持久化
- 关闭应用再打开后内容不丢失

## 5.9 右侧 AI 创作助理

对应原型区域：`ai-sidebar`。

### 功能说明

用于在当前项目和当前章节上下文中进行辅助创作。

### 首版能力

- 显示欢迎消息
- 支持快捷提示词
- 支持用户输入消息
- 支持 AI 回复消息
- 支持将 AI 结果插入正文
- 支持打开 / 关闭独立 AI 助手窗口
- 支持收起 / 展开工具抽屉

### 首版可支持的 AI 指令类型

- 描写环境
- 分析逻辑
- 润色段落
- 续写片段
- 提供剧情建议

### 交互要求

- 回车发送，`Shift + Enter` 换行
- 发送后展示用户气泡
- AI 回复采用流式输出更佳；若首版实现成本较高，可先做准实时分段返回
- 回复结束后可执行“插入光标位置”
- 当前选中文本应在输入区域附近展示，作为本次请求的附加上下文

### 验收标准

- 至少支持 1 轮真实问答
- 插入正文功能可正常工作

## 5.10 项目与应用设置

对应原型区域：`panel-settings`。

### 功能说明

用于管理模型连接、自动保存和导出行为。

### 模块 A：AI 模型配置

字段：

- 模型供应商
- API Key
- Base URL

首版能力：

- 查看与编辑
- 本地保存
- 启动时自动读取

### 模块 B：存储与备份

字段：

- 自动保存时间间隔

操作：

- 导出项目为 JSON
- 导出为 TXT

### 模块 C：主题设置

这是原型中未单独出现、但本项目首版必须补齐的设置项。

字段：

- 主题色

能力：

- 切换主题色
- 立即预览
- 本地持久化

### 验收标准

- 设置修改后应用内即时生效
- 重启应用后保留上次配置

## 7. 非功能需求

### 7.1 本地存储

- 所有项目数据默认保存在本机用户目录
- 应支持后续迁移数据目录，但首版可先固定位置

### 7.2 主题系统

- 使用统一设计令牌驱动主题
- 不允许在业务组件中散落硬编码主色

### 7.3 可用性

- 编辑器输入顺滑
- 面板切换流畅
- 滚动区域明确
- 空状态、保存成功、失败状态有可见反馈

### 7.4 桌面适配

- 面向桌面端设计，不优先适配移动端
- 最低建议宽度为三栏布局可正常显示的桌面窗口宽度

## 8. 首版不进入范围

以下功能暂不纳入首发开发：

- 角色关系图可视化
- 组织管理独立面板
- 多项目并行标签页
- 云同步
- 多账号体系
- 插件系统
- 在线社区
- 本地复杂向量记忆系统

## 9. 首版验收总标准

满足以下条件即可视为首版达标：

- 原型中的全部页面和主要交互已被一比一实现
- 项目中心、向导、世界观、角色、大纲、编辑器、设置、AI 侧栏均可真实使用
- 主题色切换可用且持久化
- 项目与章节数据可本地保存和恢复
- AI 返回结果可在编辑器中复用

## 10. 后续开发顺序建议

建议按以下顺序开发：

1. 应用骨架与主题系统
2. 项目中心与工作台布局
3. 世界观、角色、大纲、章节的本地数据结构
4. 编辑器与版本保存
5. 设置页与主题色切换
6. AI 助手接入
7. 导出能力
