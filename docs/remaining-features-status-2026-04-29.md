## 三、仍然要做的功能

## 3.1 P0：下一阶段应优先补齐

### 1. AI 上下文继续增强

当前状态：

- AI 已接入项目、章节、世界观、角色、灵感、大纲、写作风格
- 关系与组织模块已落地，但还没有系统性进入 AI 上下文主链路

剩余要做：

- 将 `organizations` / `characterRelationships` / `organizationMemberships` 纳入章节助理上下文
- 让章节续写、润色、灵感生成、角色生成能够参考角色关系与阵营归属
- 为“关系驱动冲突建议”“阵营视角改写”预留快捷动作

价值：

- 这是刚做完“关系组织模块”后的最自然延伸
- 做完后 AI 建议会更像项目内助手，而不是通用写作聊天

### 2. 章节阅读模式

当前状态：

- 章节创作页已经较完整
- 但还没有独立“阅读模式 / 成稿浏览模式”

剩余要做：

- 只读沉浸式阅读布局
- 隐藏编辑工具后的成稿浏览视图
- 章节切换时的阅读流体验
- 适合校对的字号、宽度、留白和分页策略

价值：

- 这是 `gap-analysis-vs-mumu.md` 中仍然明确缺失的能力
- 对成稿检查、节奏复读、章节通读很有帮助

### 3. 章节 AI 工作流产品化

当前状态：

- 已有 AI 对话、续写、润色、章节分析
- 但更多还是“通过助手触发能力”，还不是完整工作流

剩余要做：

- 独立“章节生成”入口
- 独立“局部改写”流程
- 基于分析结果的一键改稿链路
- 更明确的任务型入口，例如：
  - 生成本章初稿
  - 重写选中段落
  - 强化冲突
  - 调整节奏

价值：

- 这部分是对标 MuMu 时最重要的差距之一
- 能把 AI 从“聊天工具”升级为“创作流程工具”

### 4. 导入导出继续加固

当前状态：

- 已有完整项目导入导出
- 已有角色 / 大纲 / 灵感 / 关系组织 / 章节等分模块导出
- 已有一定导入校验

剩余要做：

- 更严格的 schema 版本控制
- 导入时的兼容策略说明
- 缺字段修复策略
- 模块级导入
- 冲突处理策略：
  - 覆盖
  - 新建副本
  - 合并导入

价值：

- 现在已经够用，但还不够“稳定可迁移”
- 这会直接影响长期真实使用

## 3.2 P1：重要但可以排在 P0 后

### 5. 概览页继续增强

当前状态：

- 概览页已有基础统计与聚合入口

剩余要做：

- 最近编辑章节
- 最近新增角色 / 灵感 / 组织
- 风险提醒或创作阻塞提示
- 更强的项目健康度卡片

### 6. 搜索中心升级

当前状态：

- 已有项目级统一搜索
- 已接入世界观、角色、关系组织、灵感、大纲、章节

剩余要做：

- 独立搜索中心视图
- 高级筛选
- 搜索结果排序策略
- 最近搜索历史
- 结果高亮与更多跳转上下文

说明：

- 这个能力不能算“没做”，应归类为“已完成基础版，待继续增强”

### 7. 关系图可视化

当前状态：

- 关系组织的数据结构和 CRUD 已经具备

剩余要做：

- 角色关系图谱视图
- 组织与角色的连接可视化
- 节点筛选
- 关系强度视觉映射
- 点击图节点跳回资料卡

说明：

- 现在数据层已经准备好了，后面做图不会返工

### 8. 伏笔 / 线索管理

当前状态：

- 当前还没有独立伏笔模块

剩余要做：

- 伏笔条目数据结构
- 埋点章节 / 回收章节关联
- 状态管理：
  - 已埋下
  - 待回收
  - 已回收
- 与章节分析、关系组织联动

### 9. Prompt 模板体系轻量版

当前状态：

- 已有 prompt 代码层封装
- 但没有用户可配置的模板管理界面

剩余要做：

- 内置 prompt 模板列表
- 项目级 prompt 策略
- 快捷动作模板化
- 常用任务模板复用

说明：

- 这不等于完整“Prompt 工坊”
- 先做本地轻量版更合理

## 四、暂不建议现在做的功能

这些能力价值高，但不适合当前阶段优先投入：

- MCP 插件系统
- 完整 Prompt 工坊社区化能力
- 用户体系 / 登录 / OAuth
- 管理员模块
- 在线社区
- 云同步

原因：

- 它们更偏平台化，不是当前 CharacterArc 最短板
- 当前更应该继续补强单机创作链路

1. AI 上下文接入关系与组织模块

3. 章节 AI 工作流产品化
4. 导入导出继续加固
5. 概览页增强
6. 搜索中心升级
7. 关系图可视化
8. 伏笔管理
9. Prompt 模板体系轻量版
10. 快捷键 / 日志 / 自动更新等桌面能力

## 七、结论

按当前代码状态看，CharacterArc 已经不再是“只有原型和基础 CRUD”的阶段，而是已经进入：

- 核心创作闭环已完成
- AI 创作链路已可用
- 数据层已稳定
- 下一阶段重点转向“更深的创作辅助”和“更成熟的桌面工具能力”

当前真正最值得优先推进的，不再是继续铺基础面板，而是：

- 让 AI 更理解项目内部关系
- 让章节工作流更产品化
- 让导入导出和桌面能力更稳定
