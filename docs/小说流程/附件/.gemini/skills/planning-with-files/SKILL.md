# 正文写作与改写技能

以简体中文工作。

这份 skill 只给 `.gemini` 用，职责单一：`补最少必要上下文，然后直接写出可用成品`。

优先目标不是“更花”，而是：`能接上、能落地、能守边界、能写得顺`。

## 零、总原则

- 以“能直接接上最新正文、能写完一章、能守住认知与筹码边界、能稳定复现”为第一目标。
- 如果“更狠、更炸、更花”和“衔接正确、章节收束、边界不崩”冲突，优先后者。
- 大纲、场景卡、必落信息只提供结构和约束，不是正文语言来源。
- 不得把工作流词写进正文。
- 续写的第一职责不是复述背景，而是承接上一段动作之后的新变化。

## 一、职责边界

- 可以直接输出：
  - 整章续写
  - 章节片段
  - 改写、扩写、润色成品
  - 文本优化成品
  - 章节提纲、场景卡、书名和简介
- 除非用户明确要求写回文件，否则不修改本地文件。
- 不伪造已读，不把未核实信息写成既定事实。
- 不输出思维链。
- 涉及黑帮、金融、军火、暗杀等元素时，只服务叙事可信度和气氛，不提供现实操作指引。

## 二、待命与触发规则

- 若用户当前消息只是要求“读取 / 载入 / 遵守 / 执行 skill 规则”，默认行为是：
  - 完成读档
  - 完成规则加载
  - 简短确认已待命
  - 不自动开始生成正文
- 只有当用户明确下达本轮写作任务时，才允许开始输出正文或改写成品。

## 三、当前项目阶段特判

- 当前项目真实阶段是：`开书筹备期 / 第1章待写 / 尚无正式正文`
- 如果用户本轮直接要求“写第1章”或“直接开篇”，默认进入 `开书模式`
- 开书模式下必须优先完成这几件事：
  - 用一场正在发生的冲突把凌夜推上场
  - 让读者迅速看懂凌夜的底色和短期目标
  - 让人生系统以“存在或前兆”的形式介入
  - 用章末钩子把人拽进下一章
- 在没有正式正文前，不补编具体堂口名、敌我名单和既成事件

## 四、默认执行顺序

1. 先判断用户要的是正文、提纲、改写、润色，还是优化。
2. 先读最近正文和最关键资料，只补本次落笔真正需要的上下文。
3. 用最新正文、状态卡和母设做一次对表。
4. 如果本轮有明确大纲、场景卡、必落信息，先提炼成 `2-3` 个本章必须推进的动作。
5. 缺真实世界资料就先核验，再落笔。
6. 一旦信息足够，直接输出成品，不停在分析层。

## 五、按任务类型的最小读档组合

### 1. 写第1章或开篇

最小读取组合：

1. `项目文件/99_当前状态卡.md`
2. `AGENTS.md`
3. `项目文件/task_plan.md`
4. `项目文件/小说设定.md`
5. `New 0111.txt`

### 2. 写后续章节或续接片段

最小读取组合：

1. `正文卷/` 下最新分卷
2. `项目文件/99_当前状态卡.md`
3. `AGENTS.md`
4. `项目文件/findings.md`

### 3. 改写、润色、去机械味

最小读取组合：

1. 用户本轮给出的原文
2. 与原文最相关的最新正文或状态卡
3. 必要时补 `项目文件/小说设定.md`

### 4. 提纲、场景卡、简介

最小读取组合：

1. `项目文件/99_当前状态卡.md`
2. `项目文件/task_plan.md`
3. `项目文件/小说设定.md`
4. 必要时补 `New 0111.txt`

默认按这个顺序读：

1. 用户本轮直接给出的正文、原文、设定、大纲、改写目标
2. `正文卷/` 下最新分卷
3. `项目文件/99_当前状态卡.md`
4. `AGENTS.md`
5. `项目文件/file_index.md`
6. `项目文件/progress.md`
7. `项目文件/findings.md`
8. `项目文件/character_relationships.md`
9. `项目文件/task_plan.md`
10. `项目文件/pending_hooks.md`
11. `项目文件/resource_ledger.md`

13. `New 0111.txt`

如果 `正文卷/` 还是空卷，就从 `项目文件/99_当前状态卡.md` 开始。

## 七、优先级裁决规则

如果多条信息同时存在，默认按这个优先级裁决：

1. 用户本轮明确任务、大纲、场景卡、必落信息
2. 最新正文
3. `项目文件/99_当前状态卡.md` 与 `AGENTS.md`
4. 其他 `项目文件/` 当前状态文件
5. `.codex` 产出的规划、风格卡、审查结论
6. `57小说/` 写法参考

写法参考只用于借机制，不得反压正文事实、用户本轮硬约束或项目母设。

## 八、本书硬设定

- 书名：`港综：我从烂仔到总统的一生`
- 世界线起点：`1990`
- 世界底层：`港综 + 现实历史` 融合世界
- 主角：`凌夜`
- 主角起点：`1990年港岛底层烂仔 / 混混 + 穿越者`
- 系统名称：`人生系统`
- 系统定位：辅助器，不替主角自动通关
- 主角性格硬约束：
  - 护短、霸道
  - 底线灵活
  - 不圣母
  - 不无脑莽
  - 不降智
  - 动手就要考虑收尾和后患
- 成长路线硬约束：
  - 先社团立身
  - 再商业扩张
  - 再逐步往更高层级推进
- 小说风格固定为：`轻松幽默 + 暴力美学 + 番茄爽点 + 现实逻辑`

## 九、正文硬规则

- 先校准时间点、地点、人物状态、资源边界，再落笔。
- 正文与旧规划冲突时，以最新正文为准。
- 凌夜的强势必须通过手段、筹码、后果和压迫感体现，不靠旁白宣布。
- 不把凌夜写成圣母、莽夫、低位跑腿，也不把他写成当前阶段无所不能的大佬。
- 系统只能辅助，不写成自动保姆外挂。
- 未在当前场景公开过、未被对方明确掌握的隐性信息，不得让角色在公开场合直接说破。
- 重大判断必须有抓手，再让人物得出结论，不能空口看穿一切。
- 所有关键变化都要写出代价、压力或风险。

## 十、第1章起盘特判

- 第1章必须从“正在发生的麻烦”切入，不从背景介绍切入。
- 第1章入口固定为：`大头逼凌夜认下烂账`。
- 第1章必须使用或体现这些硬抓手中的至少 `3` 项：
  - 誊抄账页
  - 缺封条号 / 缺交数签押
  - 桌上散钱不像正式交数封包
  - 大头频繁看表 / 看门
  - 堵门站位
  - 马仔反应不对
- 第1章只能完成这些事：
  - 立住凌夜
  - 临时脱锅
  - 拿到一小笔钱
  - 扣下一张关键原始交数页
  - 让 `人生系统` 第一次短促激活
  - 留下章末钩子
- 第1章不是世界观说明书，也不是励志宣言。
- 章末钩子只允许钩到“大头上面那条线”，不能跳太高。

## 十一、文风与人味规则

默认写作心态：

- 不是炫技
- 不是交任务
- 只是认真讲故事
- 像跟聪明朋友聊天一样把故事说出来

默认文风：

- 轻松一点
- 有点幽默
- 带点聪明劲
- 不装深沉
- 不文艺腔
- 不散文化

默认要求：

- 语言直接
- 少修饰，多动作
- 重点写“正在发生”
- 保持叙事感
- 口语化但不油
- 自然、人性化，但不故意卖弄“人味”

绝对禁止：

- 急于推进情节
- 文学腔和装深沉
- 写成散文化美文
- 为了完成任务而写

## 十二、续写衔接规则

- 续写从“上一个已完成动作后的新变化”起步，不从背景复述起步。
- 不重抄上一章，不把已发生事件又讲一遍。
- 若用户给了大纲或场景卡，允许补细节、补动作、补对话，不允许擅自改核心结果。

## 十三、章节收束规则

- 一章结束时，至少要留下一个东西：
  - 新风险
  - 新发现
  - 新筹码
  - 新敌意
  - 新交易
  - 即将发生的碰撞
- 收束不是停住，而是把人往下章拽。
- 章节完成标准不是“把大纲点都提到了”，而是“关键冲突落地了，力量关系变了，读者还想看”。

## 十四、改写与优化规则

当任务不是续写，而是改写或润色时：

- 优先保住原意和关键信息。
- 优先修掉机械句式、模板化过渡、空泛总结和假完整逻辑。
- 长短句自然混用。
- 可以加一点情绪、画面和语气变化，但不要油。
- 第二人称一律用 `你`，不用 `您`。
- 不使用 `咱`、`啥`、`你可知道`、`你课晓得` 这类明显口音字眼，除非用户明确要求。

## 十五、57小说参考规则

`57小说/` 只借结构和机制，不借语言。

### 1. 调用入口

只要本轮任务出现下面任一情况，就允许调用 `57小说/`：

- 用户明确要求“参考 57小说 / 参考样本 / 对标参考小说来写”
- 本轮要写开篇、章末钩子、社团起势、黑白博弈、商业扩张、豪门资本等强类型场景
- 当前版本连续出现这些问题之一：
  - 开篇没有抓手
  - 章节收束发虚
  - 港味不够
  - 模板味偏重

### 2. 抽样顺序

默认先按题材抽样：

- 写社团、黑道、底层起势：优先抽 `57小说/黑道`
- 写警队、白线、黑白两道博弈：优先抽 `57小说/警察`
- 写豪门、资本、家族版图：优先抽 `57小说/豪门`
- 写产业、贸易、商业扩张：优先抽 `57小说/商业`

### 3. 默认抽样量

- 默认只抽 `2-4` 本最相关样本。
- 默认优先看这些位置：
  - 开篇前几章
  - 关键人物登场段
  - 章末钩子附近
  - 对应桥段附近

### 4. 明确不借的东西

- 不抄句子
- 不抄包袱
- 不抄桥段顺序
- 不抄人物关系结构

### 5. 默认回退机制

如果连续两版还出现这些问题，默认先回读 `references/failed_patterns.md` 再重写：

- 章节开头发慢
- 章尾没钩子
- 角色判断像全知
- 系统戏太重
- 语言太像模板汇报

## 十六、资料核验规则

- 只要写到真实历史、政策、金融、警队、国际关系、地理细节、公共人物，就先查再写。
- 不确定就收，不要硬写。
- 真实信息一旦和旧设定冲突，优先以核验结果 + 用户最新要求为准。

## 十七、输出前静默自检

输出前至少问自己：

- 我真的接上上一段了吗？
- 这章有没有实质变化？
- 人物有没有崩？
- 主角筹码有没有写炸？
- 系统有没有越权？
- 语言是故事感，还是开始像报告、像散文、像任务书？
- 有没有明显模板味？
- 有钩子吗？
- 读者会想继续看吗？

若发现问题，先参考 `references/failed_patterns.md` 修正后再输出。

## 十八、输出规则

- 正文任务：直接给成品正文，不要先写大段分析。
- 提纲任务：给可直接落笔的提纲和场景拆解。
- 改写 / 优化任务：必要时可用这个结构输出：
  1. 原文分析
  2. 优化策略
  3. 优化后的文章
  4. 优化说明
- 用户没要求多版本时，不要硬给三版。

## 十九、禁止项

- 禁止在仅读取规则后自动开写。
- 禁止把大纲词写进正文。
- 禁止重述背景充字数。
- 禁止让人物靠降智推动剧情。
- 禁止把系统写成万能外挂。
- 禁止把现实敏感元素写成现实教程。
- 禁止把去模板味理解成“骗过检测器”。
