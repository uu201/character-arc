# Project Instruction Addendum

本文件为《港综：我从烂仔到总统的一生》的项目级硬约束补充。

## 项目定位

- 本书是 `港综 + 现实历史` 融合世界的长篇连载。
- 世界线起点固定为 `1990` 年。
- 当前真实阶段固定为：`开书筹备期 / 第1章待写 / 尚无正式正文`。
- 当前默认风格固定为：`轻松幽默 + 暴力美学 + 番茄爽点 + 现实逻辑`。
- 禁止混入玄幻、灵异、修仙、末世、科幻系统大战等超规格体系。

## 当前项目硬设定

- 书名固定为《港综：我从烂仔到总统的一生》。
- 主角固定为 `凌夜`。
- 主角起点固定为 `1990年港岛底层烂仔 / 混混 + 穿越者`。
- 系统名称固定为 `人生系统`。
- 系统定位固定为：辅助器，不替主角自动通关。
- 当前不得把系统写成“万能说明书”或“自动保姆外挂”。
- 在正式正文落地前，不得把系统后续模块、商城、数值、点数、长期奖励写成已到账事实。

## 主角硬约束

- 凌夜要护短、霸道、底线灵活。
- 凌夜不能圣母，不能无脑莽，不能为了爽点强行降智。
- 凌夜动手必须考虑收尾、后患和责任链。
- 凌夜的强势要通过判断、抓手、筹码、后果和压迫感体现，不能只靠旁白宣布。
- 凌夜的成长路线固定为：`先社团立身 -> 再商业扩张 -> 再逐步往更高层级推进`。

## 开书期硬口径

- 第1章入口已锁定为：`顶锅 / 背锅死局`。
- 第1章第一压力源已锁定为：`大头逼凌夜认下烂账`。
- 第1章场景重心固定为：`逼仄空间、桌面账本、散钱、堵门站位、即时压迫`，不靠大段时代背景介绍起手。
- 第1章只能完成这些阶段性成果：
  - 让凌夜临时脱锅
  - 拿到一小笔第一资金
  - 扣下一张关键的原始交数页
  - 让 `人生系统` 第一次短促激活
  - 抛出大头上面那条线
- 第1章禁止一口气写成大上位、大接盘、大建班底、大白线通关。

## 第1章审查红线

- 大头不能蠢到把真钱、真账和全部把柄同时摆在桌上等人拿。
- 凌夜不能“刚穿越就看穿一切”，判断必须有硬抓手。
- 硬抓手默认优先使用：
  - 账本是誊抄页，不是原始交数页
  - 缺少封条号或交数签押
  - 桌上散钱不像正式交数封包
  - 大头频繁看表或看门
  - 马仔反应与站位不对
- 控场必须来自责任链、筹码和现场压迫，不靠空口王霸之气。
- 系统第一次奖励必须克制、实用、短提示，不得长篇说明。
- 章末钩子只允许钩到“大头上面那条线”，不能一跳跳到过高层级。

涉及正文续写、改写、审查、规划、接力、维护时，默认读取顺序统一为：

`用户本轮任务` -> `正文卷/卷1_001-020.md` -> `项目文件/99_当前状态卡.md` -> `AGENTS.md` -> `项目文件/file_index.md` -> `项目文件/progress.md` -> `项目文件/findings.md` -> `项目文件/character_relationships.md` -> `项目文件/task_plan.md` -> `项目文件/pending_hooks.md` -> `项目文件/resource_ledger.md` -> `项目文件/小说设定.md` -> `New 0111.txt`

如需提炼 `57小说` 写法，只直接读取 `57小说/` 当前目录，不再引用不存在的旧清单文件。

## `.codex` 与 `.gemini` 分工

- `.codex` 负责分析、规划、审查、资料核验和项目文件维护，不默认直接交整章正文。
- `.gemini` 负责在完成最小必要读档后，直接写出正文、改写或润色成品。
- 两边必须共享同一套项目事实锚点、读档顺序和阶段口径。

- 如果核验结果与旧设定冲突，以核验结果和用户最新要求为准。
