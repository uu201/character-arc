# Changelog

本项目所有显著变更都会记录在此文件中。

格式遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [1.10.0] - 2026-06-16

### Added

- **推理模型思考过程实时输出**：针对 mimo-v2.5-pro、DeepSeek-R1、智谱 GLM-Z1、Kimi-thinking 等推理模型，新增 SSE 流拦截器自动解析 `reasoning_content` 字段，模型思考阶段实时输出思考内容到 UI，用户不再看到长时间空白等待
- **流式任务 JSON 自动修复**：流式生成路径（如写作备忘、章节审计）新增 normalize 失败后自动修复重试，与非流式路径保持一致的容错能力
- **AI 调用日志任务名称中文化**：补全 `chapter-memo`、`chapter-audit`、`chapter-repair` 等全部 35 种 AI 任务类型的中文显示名称，日志面板不再出现英文任务标识
- **初稿生成配置弹窗**：新增章节初稿生成配置界面，支持设置目标字数、选择参考作品、启用项目 Skills 和补充要求
- **拆书库用户可控注入**：拆书参考库改为用户在配置弹窗中手动选择注入，避免自动注入过多无关内容
- **跨章节写作日志 Skill**：新增跨章节写作日志和叙事功能 Skill 匹配，章节初稿生成后自动记录技法决策和效果

### Changed

- **全局助手 UI 优化**：优化全局助手界面布局与交互体验，生成提案改为按需触发
- **AI 讨论结构化写回**：支持将 AI 讨论结果结构化写回项目设定
- **章节审计与写作备忘 token 上限提升**：`chapter-audit` 和 `chapter-memo` 任务的 `max_tokens` 从 2000/4000 提升至 26000，避免长章节被截断
- **Skill 预消化机制**：章节初稿生成时自动预加载排名前 2 的技法参考文件，减少生成时的检索延迟
- **Skills 自动注入优化**：移除 tool_use 静态预判断，改为运行时根据模型能力动态决定是否使用 Skills
- **AI 写作质量提升**：新增正面技法注入、情绪弧线分析、自动修复循环和 Agent 步数提升，助手自检更严格
- **初稿生成进度浮窗隐藏**：章节初稿生成时不再显示右下角 AI 任务进度浮窗，改为在配置弹窗内展示进度

### Fixed

- **AI 调用日志项目显示**：修复日志全部显示"未知项目"的问题，补全单次任务记录
- **推理模型输出截断**：修复推理模型单次任务 `max_tokens` 不足导致结构化结果解析失败
- **流式错误静默吞掉**：修复流式错误不抛出导致请求失败时 UI 无提示
- **初稿生成与工作台白屏**：修复初稿生成完成后返回工作台时出现白屏
- **全局助手改文重复**：修复 AI 改文时内容重复插入和回车发送问题
- **拆书检索与生图兼容性**：修复 AI 助手拆书检索、token 持久化、封面调用记录与生图兼容性问题
- **工作台导航优化**：修复返回图标、隐藏小说流程面板、重排侧边栏顺序

## [1.9.2] - 2026-06-11

### Added

- **项目归档导入导出**：新增 CharacterArc 专用 `.carc` 项目归档包，导出时会从本地工作区打包项目信息、世界观、角色关系、大纲章节、流程文档、知识文档、参考作品、AI 运行记录、助手会话以及参考原文资产，便于完整备份、迁移和分享项目

### Changed

- **项目中心归档入口**：项目中心首页新增导入入口，设置页补充 `.carc` 归档导入导出操作，项目导入成功后会重新初始化项目列表并切换到导入项目
- **最近编辑时间显示**：项目卡片的最近编辑时间改为标准日期时间格式，旧的“刚刚更新 / 刚刚创建 / 刚刚导入”等占位值会统一显示为未记录，列表信息更稳定

- **全局助手 Diff 审查弹窗**：重构审查弹窗，文件卡片支持单条确认/忽略，顶部支持全部确认/忽略，底部改为动态状态计数与可折叠提醒栏，写回按钮会显示已确认项数量
- **全局助理会话与提案体验**：优化会话生成、提案链路和录入草稿引导，减少误触发与提示不清晰的情况
- **全局助理保存体验**：优化运行时保存路径，减少高频写入带来的卡顿，并改善知识文档保存提示

- **章节助手 Diff 审查交互**：修复章节助手审查弹窗交互细节，确认、忽略与写回状态更稳定
- **项目知识库展示来源**：修复项目知识库错误显示拆书知识库内容的问题
- **公告拉取兜底提示**：修复公告远程拉取失败时的提示与本地公告兜底展示

## [1.8.0] - 2026-06-05

### Added

- **全局 AI 助理独立整页**：全局助理新增独立整页入口，进入后自动隐藏工作台顶部搜索与悬浮助手入口，拥有完整的会话列表、对话流与提案写回区，适合长设定、粗大纲和项目草稿的集中整理
- **会话历史栏可收起**：整页助理左侧会话历史栏支持一键收起与展开，收起后保留新建会话与展开入口，给对话区让出更多空间
- **悬浮助手面板可调尺寸**：悬浮形态的全局助理面板支持拖拽左边缘调整宽度，并会记忆当前尺寸

### Changed

- **录入体验优化**：录入大纲 / 录入角色 / 录入历史等快捷操作改为仅填充输入框，由用户补全草稿后再发送，避免在没有草稿时直接套用已有项目内容
- **输入框自适应高度**：整页助理输入框会随内容自动增高并设有最大高度，超出后内部滚动；同时移除了暂未启用的「添加上下文」加号按钮

- **整页助理品牌图标**：空状态欢迎页的标识替换为应用自带图标

### Fixed

- **AI 调用日志迁移**：修复 AI 调用日志数据库迁移时可能重复添加列导致的报错

## [1.7.0] - 2026-06-04

### Added

- **项目级全局 AI 助理**：新增可跨世界观、角色、关系、大纲、章节、工作流文档与项目约束统一工作的全局 AI 助理，支持围绕整个项目进行问答、检索和整理
- **AI 自主调用 Skills**：全局助理提案生成接入 agent 路径，要求模型自行判断是否需要调用 `skill_load`，并按需读取项目资料后生成更贴上下文的写回方案
- **AI 调用日志入口**：标题栏模型切换区域新增 `AI 调用日志` 按钮，可查看当前项目的 AI 运行记录、模型、状态、耗时、错误信息以及工具读取次数与资料命中情况
- **工具读取轨迹记录**：AI 运行记录新增 agent `toolCalls` 轨迹持久化，让日志能真正反映全局助理读取了多少资料、走了哪些工具步骤

### Changed

- **全局助理检索策略升级**：全局助理改为更强的按需检索模式，`search_project` 支持更广范围搜索并返回 `entity_type / entity_id`，`read_project_data` 新增工作流文档、项目约束、摘要读取和精读能力
- **工具调用过程可视化**：全局助理前端会按“搜索线索 / 读取资料 / 产出动作”分组展示工具调用，让用户能看见 AI 实际读取了哪些资料、执行了哪些步骤
- **写回体验增强**：世界观、人物、大纲提案写回后会自动跳转到对应面板，并聚焦刚写回的条目，成功提示也会直接点名写回内容
- **AI 日志界面优化**：标题栏日志弹窗重构为更轻量的列表式面板，收窄宽度，保留关键信息，减少遮挡和视觉干扰
- **提案与章节工具联动优化**：普通提问不会再误入提案流程；当没有活动章节时，全局助理也会优先改走项目资料检索，而不是机械读取“当前章节”

### Fixed

- **错误提示可读性**：补齐“无活动章节”等新流程边界下的中文提示说明，避免工具链路暴露生硬英文报错

## [1.0.1] - 2026-05-25

### Changed

- **AI 接口配置**：简化为「OpenAI 兼容协议」和「Anthropic 协议」两种，用户只需选协议、填 URL 和 Key、拉取模型即可
- **Base URL 自动补全**：两种协议均自动拼接 /v1，用户无需手动添加
- **模型拉取**：统一走 OpenAI 兼容的 /v1/models 端点，自动尝试多个候选路径
- **默认模型更新**：Anthropic 默认 claude-sonnet-4-6，OpenAI 默认 gpt-4.1-mini，通义千问默认 qwen3-coder-plus 等

### Fixed

- **Anthropic 中转站连接失败**：修复 Electron 环境下 @ai-sdk/anthropic 非流式请求返回 "Invalid JSON response" 的问题，改为流式聚合
- **Base URL 路径拼接**：修复 @ai-sdk/anthropic 与中转站路径格式不兼容的问题
- **任务超时**：将 AI 任务默认超时从 90 秒调整为 300 秒，避免大模型生成时间较长时误报超时
- **非 Claude 模型兼容**：通过 Anthropic 协议使用 DeepSeek 等非 Claude 模型时，自动回退到文本生成路径

- **世界观与结构**：世界观/角色/组织/关系管理、Cytoscape 关系图谱、双栏交错时间线剧情大纲、剧情线索
- **章节创作**：VS Code 风格三栏工作区（目录树 + TipTap 编辑器 + AI 侧边栏）、自动保存与历史快照、阅读/专注模式、字数目标、`.txt`/`.docx`/JSON 导出
- **AI 辅助**：润色、续写、改写、节奏调整、摘要、伏笔识别、后续剧情链；章节初稿整章流式生成；AI 对话流式输出与中途停止；Agent Loop 循环思考与工具调用；任务进度面板
- **封面工作台**：番茄/起点/晋江/知乎盐言/七猫/刺猬猫等多平台封面 Prompt 生成与历史版本对比
- **多厂商接入**：DeepSeek（含 reasoning_content）、通义千问、智谱 GLM、Kimi、SiliconFlow、OpenAI、Anthropic、Ollama、New API / One API 网关；Embedding 模型独立配置
- **Skill 系统**：内置 15 个 Skill，frontmatter 元数据驱动，支持项目级独立导入
- **架构**：Electron + Vue 3 + TypeScript，AI 管线基于 Vercel AI SDK，导入参考小说自动建立向量索引
- **UI/UX**：Nord 风格深色 + 暖色调浅色双主题，自定义标题栏（titleBarOverlay）

### Security

- 全部数据保存在本地 SQLite，不上传任何第三方服务
