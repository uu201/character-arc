# Changelog

本项目所有显著变更都会记录在此文件中。

格式遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [1.14.2] - 2026-07-17

### Changed

- **OpenAI 兼容中转协议调整**：兼容中转默认使用 Chat Completions，只有官方 OpenAI 接口使用 Responses API，降低长文本流式请求无法收尾的概率
- **世界观分类中文化**：世界观生成、增强、项目初始化和螺旋生成统一将英文分类归一为“地理 / 法则 / 物种 / 势力 / 历史”等中文分类，并按批量生成所选类型兜底
- **模型思考内容隔离**：兼容 MiniMax、DeepSeek、通义、Kimi、GLM 等模型的多种 reasoning 字段，并在章节正文写入前清理常见思考标记

### Fixed

- **章节初稿流式事件竞态**：缓存 `streamId` 绑定前到达的 chunk、done 和 error 事件，避免快速响应被丢弃后初稿任务永久等待
- **AI 接口错误提示**：优先展示中转站返回的 `responseBody`、`error`、`message`、HTTP 状态码等信息，不再只显示笼统的 `Not Found` 或“AI 调用失败”
- **初稿正文污染**：修复模型将思考过程混入正文或使用 `<think>`、`<analysis>` 等标记时被写入章节的问题

[1.14.2]: https://github.com/zhouyeshan/character-arc/releases/tag/v1.14.2

## [1.14.1] - 2026-07-16

### Changed

- **OpenAI API 协议自动匹配**：OpenAI 官方接口以及 OpenAI 兼容接口中的 GPT-5、o1、o3、o4-mini 系列自动使用 Responses API；其他兼容模型继续使用 Chat Completions，兼顾新模型能力与第三方服务兼容性
- **AI 中转错误诊断增强**：错误日志新增请求 URL、请求字段摘要、响应头、请求 ID、SDK 重试判断与 cause 链，并提高响应正文记录上限；认证信息、Cookie 和提示词正文仍会脱敏或避免重复记录

### Fixed

- **全局助手 v2 分页读取**：项目资料工具支持通过 `offset` 连续翻页，并提示下一页位置；修复创作记忆等列表超过单页数量后重复读取首页、逐项读取耗尽预算并输出工具受限说明的问题
- **GPT-5 中转请求兼容性**：修复 OpenAI 兼容中转下 GPT-5 系列被强制按非推理模型发送旧 token 参数的问题，自动移除不支持的采样参数，并对中转站返回的通用临时上游错误执行一次受控重试
- **深度构建与结构化生成稳定性**：GPT-5 系列后台结构化任务改用流式传输后聚合完整结果，降低深度构建、批量生成等任务在部分中转站上的请求失败概率
- **Responses Base URL 规范化**：Base URL 支持识别并剥离 `/responses` 端点，避免用户填写完整 `/v1/responses` 时产生重复路径
- **AI 日志时间显示**：`ai-prompts.log` 改为记录带本地时区偏移的 ISO 时间，修复北京时间显示少 8 小时的问题

## [1.14.0] - 2026-07-16

### Added

- **AI 项目管理工具扩展**：全局助手 v2 新增项目资料、章节与分卷的结构化创建、修改和删除能力，操作仍会进入暂存区审阅，并在写回前检查目标与数据冲突
- **大纲 Excel 导入导出工作流**：支持导出分卷与章节大纲，并通过文件解析、字段映射、分卷确认、章节确认等步骤导入；可分别设置新增、覆盖或跳过策略，并预览最终目标位置
- **资料模块批量生成**：角色、世界观、组织关系与灵感接入统一的批量生成流程，支持数量、生成类型、补充要求、进度反馈与重名过滤；世界观生成类型支持直接输入自定义分类
- **大纲批量操作与跨卷拖拽**：大纲节点支持多选、批量设置分卷与处理策略，并可通过拖拽调整卷内顺序或移动到其他分卷

### Changed

- **章节初稿流程编排升级**：整合初稿、写作备忘、审计、去机械感与修复阶段，配置弹窗补充项目 Skills 和上下文控制，生成进度与阶段结果更清晰
- **大纲规划界面重构**：优化分卷布局、节点表格、选择工具栏和拖拽反馈，大量节点下的浏览、定位与批量整理更高效
- **核心资料界面收敛**：角色图鉴、关系组织、世界观和灵感模块改为紧凑资料库布局，补充搜索、分类、来源或角色定位筛选，并限制摘要和标签展示量
- **暗黑模式与窗口栏适配**：深色主题改用层次更清晰的中性炭灰色板，统一语义状态色，并让 Windows 最小化、最大化和关闭按钮随主题同步

### Fixed

- **大纲导入覆盖与分卷归属**：修复覆盖已有节点并调整目标分卷时产生重复分卷、跳过策略被统计为覆盖、确认分卷后章节目标被错误锁定等问题
- **大纲字段映射与导入完整性**：修复分卷名称、分卷摘要无法正确导入，以及字段预览信息过量、章节目标分卷选择规则不一致的问题
- **AI 大纲节点归属**：修复全局助手和结构化提案生成的大纲节点丢失目标分卷或写入错误分卷的问题
- **AI 生成与初稿结果稳定性**：完善角色、关系、世界观、灵感和大纲的结构化校验，修复初稿、审计与自动修复结果缺失或解析异常，并改善助手暂存内容与 Markdown 展示

## [1.13.1] - 2026-07-13

### Added

- **备份导入进度窗口**：导入项目归档时新增独立进度弹窗，展示读取、整理、合并、写库、恢复助手会话与参考资产等阶段，导入期间会锁定关闭操作，避免误关导致状态不明
- **章节创作分卷信息维护**：章节创作侧边栏支持新建、编辑和删除分卷信息；新建分卷信息时可绑定已有大纲分卷，避免章节分卷与大纲分卷割裂

### Changed

- **备份导入改为 worker 执行**：项目归档导入迁移到后台 worker，导入完成后自动刷新工作区并广播同步事件，减少大备份导入时主界面卡死
- **分卷目标字数标准化**：分卷目标字数改为仅保存纯数字，界面统一以“字”为后缀展示，AI 补全结果也会自动清洗为数字
- **大纲 AI 扩写语义调整**：剧情大纲顶部的“AI 扩写大纲”改为扩展新的分卷；单个分卷内的“AI补本卷”改为“AI扩写节点”，用于扩写该卷下的子节点

### Fixed

- **备份导入卡死问题**：修复导入项目备份时同步读写导致界面长时间无响应的问题，导入后会在必要时重新初始化并打开目标项目
- **分卷删除数据归属**：删除分卷时会阻止删光全部分卷，并将该分卷下的大纲节点与章节移动到相邻分卷，降低误删带来的结构丢失风险
- **章节新建分卷绑定**：章节创作中新建章节必须先选择所属分卷，再选择该分卷下的大纲节点，创建后的章节会正确继承分卷与大纲绑定
- **AI 扩写分卷结构校验**：分卷扩写要求 AI 同时返回标题、目标字数和摘要，并补充缺失字段提示，减少结构化结果不完整导致的失败

## [1.13.0] - 2026-07-03

### Added

- **全局助手 v2**：新增基于 Assistant Runtime v2 的全局助手体系，支持会话持久化、事件流、右侧浮层、暂存变更审阅、分批任务续跑与更清晰的工具过程展示
- **v2 审计知识沉淀**：全局助手 v2 的审计模式会将阶段结论与风险摘要保存到项目知识库，方便后续追踪设定矛盾、伏笔风险与修正依据
- **v2 自动匹配 Skills**：助手 v2 会根据当前对话、场景、项目题材与写作阶段自动匹配候选 skills；强制 skills 直接生效，普通候选按需加载
- **章节新建绑定大纲弹窗**：章节创作侧边栏的新建章节入口会先弹出大纲选择弹窗，选择大纲后自动带入章节标题，并继承摘要、字数目标和大纲绑定

### Changed

- **全局助手 v2 工具能力增强**：完善项目资料、章节、大纲、人物、世界观、组织、剧情线、创作记忆、项目约束、技能与知识库等工具读取能力，并优化对话样式与流程日志
- **暂存变更体验回归统一审阅区**：章节修改暂存展示从编辑器内联改回助手暂存区，避免修改入口分散；选区内容不再重复携带，减少局部改写跑偏
- **v2 上下文窗口大幅提升**：默认按 128k 长上下文窗口预估，项目/章节上下文可使用约 96k token；超出窗口时优先压缩保留头尾和关键信息，仍不足时才省略
- **大纲与章节创作体验优化**：大纲面板时间线、进度条、分卷标记和节点卡片样式全面优化；助手加载态、流式提示和暂存区提示文案更清楚
- **旧版全局助手入口收敛**：隐藏旧版全局助手菜单，减少新旧助手并存带来的入口混乱

### Fixed

- **v2 上下文与实体修改意图**：修复同一会话中目标实体上下文丢失、根据已有框架给建议时跑成全项目审计等问题
- **v2 大纲查询完整性**：修复大纲中存在第十三章但助手查询不到的问题；大纲索引支持序号显示和自然语言章节序号定位
- **AI 新建章节大纲绑定**：修复通过 AI 创建章节时未写入 `outline_item_id` 的问题，按大纲生成新章节会绑定对应大纲节点
- **全局助手章节正文修改能力**：修复全局助手无法稳定读取并修改章节正文的问题，并优化章节正文审阅与意图判定

- **番茄风向标切换榜单更顺滑**：切换男频/女频时改用独立的 `switching` 态而非全量 `loading`，仅在内容容器上加渐隐过渡，避免整页骨架闪烁；刷新数据时优先保留当前所选榜单，减少视觉跳变
- **番茄风向标卡片样式调优**：摘要卡片改用 `color-mix` 混色背景与主题色边框，弱化原渐变的视觉过载，与卡片层级更协调

### Fixed

- **章节编辑器查找替换若干 bug**：

  - 修复上一个/下一个不滚动到匹配位置的问题（用 `coordsAtPos` 精确定位匹配字符，手动平滑滚动编辑器容器）
  - 修复点击「替换」抛出 `Applying a mismatched transaction` 的问题（替换命令改用 TipTap 注入的 `tr.insertText`，避免 chain 与外部 dispatch 时序冲突）
  - 修复关闭搜索浮窗时编辑器跳回顶部的问题（清空命令 dispatch 前后保存并还原 `scrollTop`）
- **公告远程拉取仍展示旧缓存**：成功拉取但内容比本地兜底更旧时不再视为最新，改为 fallback 走本地公告判定 `new` / `error` 状态，避免发版后还反复弹出过期内容；同步修正 v1.6.0 历史公告中被错误剥离的书名号

## [1.11.0] - 2026-06-27

### Added

- **番茄风向标页面**：新增独立页面，支持查看番茄小说女频/男频新书榜，含 AI 风向速评、热门综合赛道卡片、热门分类、高频题材标签云、分类榜单与排名趋势侧栏，数据来源于 FanqieRankTracker 仓库，经 jsDelivr 四通道镜像加速并本地缓存 6 小时
- **书卡操作按钮**：分类榜单每张书卡底部新增「访问原文」和「复制简介」按钮，一键在浏览器打开番茄小说原文页或复制简介到剪贴板
- **热门综合赛道卡片升级**：排名由淡灰水印数字改为金/银/铜小徽章，top1 卡片加主题色渐变背景与顶部高亮条；新增 `new_count`、`dropped_count`、`active_days` 三项指标行；领涨分类改为 chip 微标 + 强调值组合，底部分类标签高亮领涨项
- **创作助理放开输出篇幅限制**：移除创作助理硬性字数限制，篇幅改由请求内容决定，`maxTokens` 仅作安全上限；章节助手空状态与续写模式新增引导提示，整章正文创作导向「生成初稿」入口
- **章节初稿接续契约**：生成本章时自动注入上一章结尾原文（而非开头预览），要求模型在时间、地点、在场人物、情绪上自然承接，禁止场景无交代跳变
- **agent 循环可靠性增强**：步数耗尽时禁用工具强制收尾，把残缺/空输出补成完整答案；新增同参数重复工具调用检测，回灌首次结果节省步数；截断时显式报错，不再静默返回空；各任务补齐独立 `maxSteps` 配置（深度拆书 16、全局助手 12 等）
- **skill 选择逻辑增强**：`required` skill 无条件保留，不再被高分 optional 在 slice 时挤掉；叙事功能匹配增加中文 triggers 维度，对中文工具箱真正生效；同分排序加 `priority/id` 保证跨次运行确定性；选择日志输出评分分项明细便于调参

### Changed

- **公告远程拉取走 CDN 加速**：公告拉取从单通道 `raw.githubusercontent.com` 改为 jsDelivr / fastly / ghproxy / raw 四镜像通道自动切换，新增 1 小时本地磁盘缓存，弱网下可回退旧缓存；抽取公共 GitHub 多镜像抓取工具模块，风向标与公告共用同一套实现
- **批量拆书弹窗状态持久化**：将批量拆书状态从组件级 ref 提升为模块级单例 composable（`useBatchImport`），注册到全局 `AiTaskProgressDock` 浮层，页面切换或最小化后不再丢失进度，可在右下角查看

### Fixed

- **story-state 角色过滤失效**：修复章节生成时 story-state 角色过滤退化为注入全项目所有角色状态的问题，补回 character id 过滤
